Git 
- Version Control System, 버전 관리 시스템

- 장점: 쉽다.
- 단점: 파일명이 매번 바뀐다.(규칙 없음)
예) 기말고사_레포트.doc > 버전(Version). 스냅샷(Snapshot)
    기말고사_레포트_최종.doc
    기말고사_레포트_최종_최종.doc
    기말고사_레포트_최종_최종_정말최종.doc > 최종 > 기말고사_레포트.doc

예) 레포트   > 기말고사_레포트.doc + 첨부파일.zip > 버전
    레포트_1 > 기말고사_레포트.doc + 첨부파일.zip > 버전
    레포트_2 > 기말고사_레포트.doc + 첨부파일.zip > 버전

- 내가 원하는 시점 > 파일과 디렉토리의 상태를 미리 저장(=버전)해 놓고 > 나중에 필요에 따라 이전 버전으로 되돌아갈 수 있도록 도와주는 시스템

Git 설정하기

1. 사용자 정보 등록하기
    - 버전을 누가 생성/관리하는지 기록하기 위해서
    - --global: 현재 사용자의 모든 Git 저장소에 설정이 적용

$ git config --global user.name <이름>
$ git config --global user.email <이메일>

$ git config --global user.name <GitHub 계정명>
$ git config --global user.email <GitHub 계정 이메일>

$ git config --global --list

$ git config --global user.name pinn.public
$ git config --global user.email pinn.public@gmail.com


2. 기본 브랜치명 바꾸기

$ git config --global init.defaultBranch main


3. 엔터 설정하기
- 협업 > 윈도우 사용자 + 맥 사용자 + 리눅스 배포
       - 윈도우(\r\n), 맥/리눅스(\n)
- Git > 커밋 발생 > 모든 엔터를 \n 변환
      > 체크아웃(다운로드) > \r\n 변환

$ git config --global core.autocrlf true


4. Git Bash에서 한글이 깨질 때

$ git config --global core.quotepath false

내역 확인하기
- HEAD(현재 위치가 이 커밋입니다.) 포인터
- 옵션
    1. -p: patch, 각 커밋의 변경 내역(diff)을 함께 보여준다.
    2. --oneline: 한 줄 커밋 메시지 출력
    3. -n <숫자>: 최근 n개 커밋만 출력
    4. --stat: 수정된 파일과 변경된 행 수 요약 표시
    5. --author=<이름>
    6. -all: 모든 브랜치 조회
    7. --graph: 브랜치 현황을 그래프로 출력

특정 커밋을 지정하는 방식
$ git checkout <커밋 ID>
$ git checkout main # 최식 커밋으로 돌아가기
$ git checkout HEAD
$ git checkout HEAD^        # 이전 커밋
$ git checkout HEAD^^       # 2단계 전 커밋
$ git checkout HEAD^2       # 2단계 전 커밋
$ git checkout HEAD~        # 이전 커밋
$ git checkout HEAD~1       # 이전 커밋
$ git checkout HEAD~2       # 2단계 전 커밋

- 과거 시점의 특정 커밋으로 이동해서 > 그 커밋 이후의 커밋을 삭제한다.

혹시 > reset 하기 전으로 돌아가기

revert 명령어

merge 명령어
- 서로 다른 두 브랜치의 변경 사항을 하나로 합치는 명령어
- 기능 브랜치와 원 브랜치를 하나로 합칠 때

1. fast-forward merge
    - 원 브랜치에서 분기된 브랜치를 합칠 때
    - 분기된 브랜치에는 변화가 발생(새로운 커밋 발생)
    - 원 브랜치에는 변화가 없음(새로운 커밋 없음)

--------------------------------------------


1. checkout // 과거 커밋 열람 (읽기 전용)
2. reset //  어떤 특정 커밋 이후 ~ 최신 커밋을 삭제하는 용도
3. revert // 특정 커밋들만 삭제하는 용도
4. branch // 브랜치
5. merge // 병합
6. rebase // 병합

*** 협업에서 지양하는 것들
- reset > revert
- rebase > merge
- git commit --amend //커밋 ID 변경됨

$ git switch main // 성공

$ git stash save "수정하기/작업하기 작업중"

$ git status

$ git stash list

$ git switch main // 성공

$ git switch board

$ git stash list

$ git stash apply //최신 stash 적용
$ git stash apply stash@{0} //특정 stash 적용

$ git status

$ git stash list

$ git stash drop
$ git stash drop stash@{0}

$ git stash pop // apply + drop
$ git stash pop stash@{0}
